# shishka-tablered
Массовое редактирование однотипных ODS таблиц
...
## Техническое задание
Локальная программа для массового редактирования .ods-файлов с группировкой по количеству строк

### 1. Назначение и цели
Разработать локальное десктоп-приложение на Python (без веб-фреймворков, минимум зависимостей), которое:
- сканирует папку со своим исполняемым файлом и находит все ".ods"-файлы;
- анализирует каждый файл (подсчитывает количество строк на первом листе);
- выводит статистику: сколько файлов имеют определённое число строк (группировка);
- позволяет пользователю выбрать одну из групп (например, «60 файлов с 80 строками») и открыть один файл из этой группы для редактирования (образец);
- вносит изменения (значения ячеек, шрифт, цвет текста) в этом единственном файле;
- даёт возможность:
  - сохранить изменения только в этом файле (кнопка «Сохранить этот файл»);
  - сохранить изменения во всех файлах текущей группы (с тем же количеством строк) — кнопка «Применить ко всем однотипным ODS».

// Ключевой результат: пользователь не открывает каждый файл вручную — он правит один образец, а программа автоматически переносит правки на все файлы с идентичной структурой (одинаковое число строк). Экономия времени при работе с сотнями таблиц.

### 2. Функциональные требования

#### 2.1. Сканирование и группировка

- При запуске программа получает путь к своей директории (там, где лежит ".exe" или ".py").
- Рекурсивно или только в корне папки? – Только в корне папки (без вложенных папок, если не сказано иное), либо с возможностью выбора папки через кнопку «Выбрать папку».
- Находит все файлы с расширением ".ods" (регистронезависимо).
- Для каждого файла:
  - открывает первый лист;
  - определяет количество строк с данными (игнорируя полностью пустые строки в конце или используя "nrows()" из библиотеки ODS);
  - если файл повреждён или не может быть прочитан — пропускает, записывает ошибку в лог.
- Формирует словарь / группировку:  
  "{ число_строк : [список_файлов] }"
- Выводит статистику в главном окне, например:

  """
  Найдено .ods файлов: 102

  Группы по количеству строк:
  • 80 строк — 60 файлов
  • 40 строк — 40 файлов
  • 39 строк — 1 файл
  • 38 строк — 1 файл
  """

#### 2.2. Выбор группы и открытие образца

- Пользователь выбирает группу (например, щелчком по строке с «80 строк — 60 файлов»).
- Появляется кнопка «Открыть образец (один файл из группы)» или двойной щелчок по группе.
- Программа берёт первый файл из списка этой группы (или позволяет пользователю выбрать конкретный файл из группы, если нужно).
- Открывает этот файл в табличном редакторе (как встроенный аналог Calc) — см. пункт 2.3.

#### 2.3. Редактирование одного файла (образца)

- Интерфейс: таблица (строки × столбцы) с возможностью редактирования ячеек двойным щелчком или через строку ввода.
- Поддерживается:
  - изменение текста / чисел / дат;
  - изменение шрифта (гарнитура, размер, жирность, курсив) для выделенной ячейки или диапазона;
  - изменение цвета текста (диалог выбора цвета).
- Кнопки / меню:
  - «Сохранить этот файл» — перезаписывает текущий открытый файл на диске (без влияния на другие).
  - «Применить ко всем однотипным ODS» — см. раздел 2.4.
  - «Закрыть файл / Вернуться к статистике» — возврат к списку групп.

#### 2.4. Массовое применение (ключевая функция)

- При нажатии «Применить ко всем однотипным ODS» программа:
  1. Проверяет, какие изменения были сделаны в текущем открытом файле-образце (по сравнению с оригиналом этого же файла).  
     Для этого необходимо запоминать:
     - изменённые значения ячеек (координаты и новое содержимое);
     - изменённое форматирование (шрифт, цвет) для каждой ячейки.
  2. Запрашивает подтверждение: «Будет обновлено N файлов (все с ХХ строками). Создать резервные копии?» (рекомендуется «Да»).
  3. Для каждого файла в текущей группе (все файлы с таким же количеством строк):
     - загружает файл;
     - применяет к нему те же изменения (изменяет те же координаты ячеек на те же значения, устанавливает те же шрифты/цвета);
     - сохраняет файл (с заменой оригинала или с перезаписью, после резервного копирования).
  4. Показывает прогресс (какой файл обрабатывается, сколько осталось).
  5. По окончании выводит отчёт: сколько файлов успешно обновлено, сколько с ошибками (если структура не совпадает — например, недостаточно строк, но по логике группы число строк одинаковое, ошибка маловероятна, но возможна при повреждении).

- Важно: массовое применение не требует открытия этих файлов пользователем. Всё происходит в фоне.


#### 3.2. Зависимости и стек
- Python 3
- Библиотеки:
  - "ezodf" (легковесная, поддерживает стили шрифтов/цветов) или "odfpy" + "lxml".
  - "tkinter" (встроенный) — для GUI.
  - numpy и др. библиотеки - при необходимости
- Без фреймворков
- Распространение: .py скрипт с баш-скриптом устанавливающем питон и все зависимости (нужно учесть что эта программа
может быть запущена на компьютере без питона вообще и впервые; баш-скрипт должен все доставить через apt install)

#### 3.3. Удобство и простота
- Интерфейс не должен выглядеть как IDE. Всё просто:
  - список групп с количеством файлов;
  - кнопка «Открыть образец»;
  - редактор таблицы;
  - две главные кнопки («Сохранить этот файл» / «Применить ко всем однотипным»).

#### 3.4. Надёжность
- При ошибке чтения любого файла (кроме образца) — запись в лог, программа не падает.
- При массовом применении, если какой-то файл не удалось сохранить (нет прав, повреждён), программа продолжает с остальными.
- Все изменения в образце до нажатия «Сохранить» удерживаются в памяти, исходный файл не трогается.

### 5. Ограничения и допущения
- Поддерживается только первый лист каждого ODS-файла.
- Однотипность — только по количеству строк. Не проверяется совпадение количества столбцов или структуры. Это на совести пользователя.
- Формулы: при замене содержимого ячейки формула удаляется, остаётся только новое значение (поведение как в Calc при ручном вводе).
- Цвет и шрифт применяются к тексту ячейки, не к фону. При желании можно расширить, но в рамках текущего ТЗ это не требуется.

### 6. Критерии приёмки

1. Программа правильно подсчитывает число строк во всех ".ods" файлах в выбранной папке (без пустых строк в конце).
2. Группировка отображается точно (файлы с одинаковым количеством строк объединены).
3. Пользователь может открыть любой один файл из любой группы и отредактировать:  
   - содержимое любой ячейки;  
   - шрифт (жирный, курсив, размер);  
   - цвет текста. 
	- может отредактировать несколько разных ячеек в образце и применить их одновременно
4. Кнопка «Сохранить этот файл» сохраняет изменения только в том файле, который открыт.
5. Кнопка «Применить ко всем однотипным» изменяет все файлы из той же группы (с тем же количеством строк) идентичным образом (те же ячейки, те же значения, шрифт, цвет).
6. Перед массовым применением создаётся резервная копия всех изменяемых файлов.
7. Программа работает без интернета, без установки дополнительных библиотек в коде самого .py (дял установки зависимостей делай отдельный баш-скрипт, который запускается 1 раз при установке)
8. Рекурсивное сканирование вложенных подпапок не требуется; предполагается что юзер копирует .py файл каждый раз в ту папку, с которой
собирается работать, предварительно перебросив туда все ODS файлы и разумеется самостоятельно сделав бэкапы этих ODS файлов.
программа резервным копированием и рекурсивным сканированием не занимается. максимальный минимализм
